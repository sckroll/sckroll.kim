# 병렬 트랙 — 대표 콘텐츠 준비

## 트랙 상태

- 상태: 병렬 진행
- 개발 차단 여부: Phase 06까지 비차단
- 공개 배포 차단 여부: Phase 07 완료 전 필수
- 개발 대체 수단: 도메인 계약 기반 더미 fixture

## 사용자 체크리스트 — 프로젝트

- [ ] 공개 후보 프로젝트 2~3개 선정
- [ ] 프로젝트별 NDA·저작권·보안 정책 확인
- [ ] 회사 승인 필요 여부 확인
- [ ] 프로젝트명 공개·익명화 방식 결정
- [ ] 한 줄 설명 작성
- [ ] 해결한 문제와 사용자 맥락 작성
- [ ] 본인 역할과 책임 범위 작성
- [ ] 주요 제약과 의사결정 3개 이상 작성
- [ ] 정량·정성 성과 작성
- [ ] 기술 스택과 선택 이유 작성
- [ ] 사용 가능한 이미지·영상 정리
- [ ] 민감 정보 제거 여부 최종 확인

## 사용자 체크리스트 — 포스트

- [ ] 초기 공개 포스트 1개 이상 선정
- [ ] 기술·커리어·제품·일상 중 분류 결정
- [ ] 제목과 한 줄 요약 작성
- [ ] 대표 이미지 또는 이미지 없는 레이아웃 결정
- [ ] 태그와 게시 일시 정리
- [ ] 외부 링크와 인용 출처 검수
- [ ] 초안·공개 상태 구분

## 사용자 체크리스트 — 이력서와 소개

- [ ] 3문장 이내의 자기소개 작성
- [ ] 프론트엔드 핵심 강점 3개 정리
- [ ] Product Engineer 확장 경험 정리
- [ ] 경력 기간과 역할 정리
- [ ] 기술 역량의 숙련도 표현 방식 결정
- [ ] 공개 가능한 교육·자격·수상 정보 정리
- [ ] 이메일·GitHub·외부 프로필 확인
- [ ] 웹 이력서와 PDF 이력서의 내용 차이 검수

## 사용자 체크리스트 — 미디어

- [ ] 프로필 이미지 사용 여부 결정
- [ ] 프로젝트별 커버 이미지 선정
- [ ] 이미지 저작권과 공개 권한 확인
- [ ] 영상 자막과 대본 준비 여부 확인
- [ ] Blender 결과물의 웹용 파생본 준비
- [ ] 원본 파일의 웹 배포 제외 확인
- [ ] 대체 텍스트 초안 작성

## 에이전트 지원 체크리스트

- [ ] 프로젝트 상세 작성 템플릿 제공
- [ ] 익명화와 민감 정보 제거 체크리스트 제공
- [ ] 포스트 구조와 요약 초안 지원
- [ ] 경력 문장의 채용 담당자 관점 편집 지원
- [ ] 이미지 대체 텍스트 편집 지원
- [ ] 실제 콘텐츠의 Notion 스키마 적합성 확인
- [ ] 실제 콘텐츠 교체 전 더미 fixture와 필드 매핑 확인

## Phase별 연결 지점

- Phase 01: 실제 콘텐츠 없이 더미 계약 기준의 스키마 확정 가능
- Phase 02: 실제 길이를 모사한 더미 제목·요약·미디어로 디자인 검증 가능
- Phase 04: 더미 fixture 기반의 전체 사용자 흐름 구현 가능
- Phase 05: 비공개 또는 빈 Notion 결과와 더미 adapter의 병행 검증 가능
- Phase 06: 실제 콘텐츠 없이 접근성·성능·검색 기술 검증 가능
- Phase 07: 공개 대상 더미 콘텐츠의 실제 콘텐츠 교체 필수

## 완료 조건

- [ ] 공개 프로젝트 최소 2개 준비
- [ ] 공개 포스트 최소 1개 준비
- [ ] 소개와 경력 정보 준비
- [ ] 공개 미디어와 대체 텍스트 준비
- [ ] NDA·저작권·보안 검수 완료
- [ ] `POSTS`와 `PROJECTS`의 실제 입력 완료
- [ ] 공개 환경의 샘플 표기와 더미 데이터 제거
